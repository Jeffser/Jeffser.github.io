Attachments

Attachments are a way of appending additional data to Messages, both to give information to the user and to add context to a Chat.

Global Attachment Button

Used by the Global Footer to select attachments, if pressed it will prompt a file selector, if right clicked / long pressed, it will show a menu with multiple options.

A modified version without the menu is used in the Model Creator, to add files to the context.

Menu

  • Attach File: The default behavior, it opens a file selector.
  • Attach Website: Shows a dialog to input a Website URL.
  • Attach YouTube Captions: Shows a dialog to input a YouTube Video URL.
  • Attach Screenshot: Sends a request for the system to take a screenshot and send it to Alpaca.
  • Attach Photo From Camera: Opens the Camera Activity

Styles

Alpaca has two ways of presenting attachments in the Global Footer and Messages.

Simple

This is the one used almost everywhere with the exception of Images in Messages.

Additionally this style will be used when attaching an Image to the Global Footer.

Image

This is only used when a Message contains an image.

Types

Alpaca uses different types of attachments to represent data, some of them are added to the context of the Chat and some are only added in order to give additional information to the user.

Document

The generic type used to represent extracted data from a list of compatible document types.

Document types:

  • Plain Text (.txt, .md)
  • PDF (.pdf)
  • Open Document Text (.odt)
  • Microsoft Word Document (.docx)
  • Microsoft PowerPoint Presentation (.pptx)

Used by:

  • User

Ways of attaching:

  • When attaching a document file manually.
  • When using the Attachment Creator Activity.

Code

Used to represent code files provided by the user, they don't specify the language by themselves but the name of the file is shared with the Added Model when used in a Chat.

Code file types:

  • C / C++ (.c, .h, .cpp, .cxx, .cp, .hxx, .inc)
  • Web (.html, .css, .js, .ts)
  • Python (.py, .pyw)
  • Java (.java)
  • C# (.cs, .csx)
  • Data (.json, .xml, .yaml, .csv)
  • Assembly (.asm, .nasm)
  • Lisp (.lsp, .lisp, .el, .emacs)
  • Compute (.cu, .glsl)
  • Scripting (.lua, .php, .rb, .sh)
  • Rust (.rs)
  • Database Script (.sql)
  • Container (.dockerfile)
  • PICO-8 (.p8)
  • Any unrecognized plain text file

Used by:

  • User

Ways of attaching:

  • When attaching a code file manually.

YouTube

Used when the user attaches a YouTube video, it specifies the title, description and transcript of the given video.

Warning

This feature might break sometimes because of YouTube's backend changes, please check output before sending messages.

Used by:

  • User

Ways of attaching:

  • Select Attach YouTube Captions from the Global Attachment Button's Menu.
  • Paste a YouTube video link in the Message Entry.
  • Select Attach option in Web Browser Activity.

Website

Used to share the extracted data from the Web Browser, these attachments are usually really big and should only be used with large Added Models.

Warning

This feature is dependent on the website structure, additionally, some websites might have anti scrapping functions that might prevent data extraction, please check output before sending messages.

Used by:

  • User
  • Tool

Ways of attaching:

  • Select Attach Website from the Global Attachment Button's Menu
  • Paste a website link (other than a YouTube link) in the Message Entry
  • Select Attach option in Web Browser Activity.
  • Using the Web Search Tool.

Thought

Used to check the thought process from compatible Added Models after the Thinking Block is removed.

Used by:

  • Model

Ways of attaching:

  • Automatically done by model when it finishes a thought process.

Tool

Used to share the results of Activities in Messages.

Used by:

  • Tool

Ways of attaching:

  • Automatically done by tool when needed.

Link

Used to share links to websites in messages, it does not extract data from the website, only links to it. The main use for it is when using an Ollama Added Model that requires cloud authentication for the first time.

Used by:

  • Model

Ways of attaching:

  • Automatically done by model when needed.

Images

Used to represent images inside the chat, please note that the Added Model in use needs to have the vision capability to be able to interpret the image, otherwise it will just ignore it.

File types:

  • Raster / Bitmap (.ani, .bmp, .gif, .jpeg, .png, .pnm, .tga, .tiff)
  • Vector (.svg, .wmf)
  • Icon (.icns, .ico, .ani)
  • Toolkit Specific (.qtif, .GdkPixdata)

Used by:

  • User
  • Tool
  • Activity

Ways of attaching:

  • When attaching an image file manually.
  • When using the Background Remover Tool as the result.
  • When using the Camera Activity

Audio

Used to represent transcripts of audio files (or audio from video files), the models will only see the transcript and won't have clues of tone or melody of audio.

File types:

  • Audio (.wav, .mp3, .flac, .ogg, .oga, .m4a, .aac, .aiff, .aif, .opus)
  • Video (.webm, .mp4, .mkv, .mov, .avi)

Used by:

  • Activity

Ways of attaching:

  • When attaching an audio or video file (it will open the Transcriber Activity)
  • When using the Transcriber Activity directly

Metadata

Used to represent the metadata of a response from an Added Model of an Ollama Instance with the option Show Response Metadata turned on in the Instance Preferences Dialog.

Used by:

  • Model

Ways of attaching:

  • Automatically done when the model shares the information.

Viewing Content

When the user clicks an attachment it will open one of two different viewers.

  • File Viewer
  • Image Viewer
