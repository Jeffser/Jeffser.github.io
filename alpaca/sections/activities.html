Activities

Activities are mini-apps integrated with Alpaca, they have many uses like adding Attachments, displaying Tools actions and interacting with Blocks.

Display Modes

Activities have a few ways of displaying themselves depending on the context and selected option in Preferences

Sidebar

Used in the Main Alpaca Window, it displays the activities on the right side of the app, it can also show the Activity Launcher

Features:

  • Activity Launcher
  • Tab Management
  • Detach Activity Option
  • Bottom Action Bar

Window

An alternative way of using activities, the user can use it by detaching a Sidebar activity or by selecting the window option in the Preferences Dialog

Features:

  • Activity Launcher
  • Tab Management
  • Attach Activity Option
  • Bottom Action Bar

Dialog

This is a legacy option used by the code runner, file viewer and other activities before the concept of activities was created, it can still be used by selecting the dialog option in the Preferences Dialog.

Even though it is considered legacy and might limit the usability of some activities, there aren't any plans to remove it from Alpaca, because it is the way Quick Ask displays activities.

Features:

  • Bottom Action Bar

Activity Launcher

Used to launch activities manually, it can be accessed on the Sidebar or an existing Activity Window.

Not all activities can be accessed from the launcher since some are dependent on existing content, for example the File Viewer and Image Viewer

Tab Management

Used by the Sidebar and Activity Windows to manage multiple activities at the same time.

Attaching / Detaching Activities

This is a way for activities to quickly switch between being on the Sidebar or a Window.

To use it press the attach / detach button in the respective mode.

Detach Attach

Action Bar

Used by all activities* to show controls, these can change dynamically depending on the content of the activity.

They all have in common the close activity button which is always the last bottom at the bottom right corner of the activity.

*Live Chat uses it's own bar in the form of a Global Footer Action Bar.

Available Activities

A list of all the currently available activities, all the screenshots have been taken in Window Mode.

The activities preferences are available in the Preferences Dialog.

Terminal

Used to interact with code and the device, it uses VTE as it's terminal emulator.

When running Alpaca through Flatpak it's better to use the terminal in SSH mode.

Preferences:

  • Connection type (Directly or SSH)
  • Username (SSH)
  • Device IP (SSH)

Ways of accessing it:

  • Activity Launcher
  • Used as child by Code Runner
  • Terminal Tool

Attachment Creator

Used to create new Attachments for use directly in the Global Footer's Attachment Container. It's meant to give a bigger text field for complex prompts.

Ways of accessing it:

  • Activity Launcher

Camera

Used to capture an image directly inside of Alpaca.

Ways of accessing it:

  • Activity Launcher
  • Global Attachment Button's Menu

Web Browser

Quickly access the internet directly from within Alpaca, users can also create Website Attachments based on the contents of the current website by pressing the Attach button.

Preferences:

  • Search Engine (Startpage / DuckDuckGo / Google / Custom)
  • Search Query URL (Custom)
  • Homepage URL (Custom)

Ways of accessing it:

  • Activity Launcher
  • Web Search

Latex Editor

Latex is a language used to represent mathematical equations, it is also widely understood by most Added Models.

It is mean as a way of editing existing Latex Renderer Blocks and create new equations that can be exported as images.

Ways of accessing it:

  • Activity Launcher
  • Editing a Latex Renderer Block

Live Chat

Live Chat was first introduced as a separate window but eventually got ported into being an activity.

It allows the user to have an ephemeral chat focusing in Text to Speech and Speech to Text capabilities, meant to be used as a way of having natural conversations with Added Models.

Ways of accessing it:

  • Activity Launcher

Transcriber

Used to transcriber the user's files or voice into a new Audio Attachment for use in the Global Footer's Attachment Container, using the Speech to Text Models.

Ways of accessing it:

  • Activity Launcher
  • Attaching an audio / video file

Background Remover

Uses Background Remover Models to remove the background of an image provided by the user either directly or in Chat.

Preferences:

  • Default Model

Ways of accessing it:

  • Activity Launcher
  • Background Remover Tool

Code Editor

Used to edit Code Blocks.

Ways of accessing it:

  • Pressed the edit button of a Code Block
  • Used as child by Code Runner

Code Runner

It combines the Terminal and Code Editor, the user can make modifications to the code and test the results without saving to the Code Block.

Ways of accessing it:

  • Pressed the run button of a Code Block

File Viewer

Used to preview all the Attachment Types (except Images and Links), it also gives the option to delete and export the Attachment.

Ways of accessing it:

  • Pressed an Attachment (except Images and Links)

Image Viewer

Used to preview Images, it features zoom and panning like a normal image viewer, it also gives the option to delete and export the Image.

Ways of accessing it:

  • Pressed an Image Attachment
  • Pressed the original or result image from a Background Remover Activity
