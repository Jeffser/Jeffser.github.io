Models

Alpaca uses models to generate content, they are managed by different Instances.

Model Types

Alpaca represents models using a single basic model button widget, it can be modified by each type of model to represent different information.

Added Model

Also known as Text Generation Model, the user can modify their profile picture and even make custom children if the Added Model Instance in use is an Ollama Instance.

They show the pretty name of the model followed by it's family (if available) and tag, they can also show a profile picture if set.

Model Creator

Used by Ollama Instances to create children based on GGUF files or existing Added Models.

There are multiple ways of starting a model creation.

  • Right click an Added Model and select Create Child.
  • Click Create Child inside of an Added Model Dialog.
  • Select the Create From Existing option inside the Model Manager drop-down menu (top-right corner).
  • Select the Create From GGUF option inside the Model Manager drop-down menu (top-right-corner).

Model Manager

Allows the user to remove models and pull new Added Models, as well as changing their profile pictures and creating children if using an Ollama Instance.

In addition to deleting other types of Models.

Added Model Selector

Used by the Global Footer to select the model to use in the next response.
